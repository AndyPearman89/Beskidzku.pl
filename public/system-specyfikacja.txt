BESKIDZKU.PL — SYSTEM SPECYFIKACJI (KANONICZNY TXT)

1. OVERVIEW
Beskidzku.pl to regionalna platforma turystyczno-biznesowa działająca jako:
- portal SEO (programmatic content)
- marketplace lokalnych usług
- silnik lead generation
- warstwa SaaS dla firm (wizytówki + monetyzacja)

System jest oparty o WordPress + PearTree Core i działa jako modularny monolit z możliwością skalowania na kolejne regiony.

2. CORE CEL
Zbudowanie dominującego źródła:
- ruchu organicznego (SEO)
- danych lokalnych (noclegi, atrakcje, firmy)
- leadów sprzedażowych

Model: TRAFFIC → INTENT → LEAD → REVENUE

3. STRUKTURA PORTALU
Główne sekcje:
- Regiony
- Miejscowości
- Noclegi
- Atrakcje
- Szlaki
- Firmy
- Jeziora
- Pogoda

Każda sekcja oparta o CPT + ACF + relacje między obiektami.

Relacje:
- miejscowość → region
- nocleg → miejscowość
- atrakcja → miejscowość
- szlak → region/miejscowość
- firma → lokalizacja

4. ARCHITEKTURA TECHNICZNA
Stack:
- WordPress Multisite
- PearTree Core (DDD architecture)
- GeneratePress Child Theme
- ACF (Advanced Custom Fields)
- Leaflet (mapy)
- AJAX + REST API

Struktura:
- modularny monolit
- src/Domain / Application / Infrastructure / UI
- FeatureGate (pakiety)
- Tenant-aware (multisite ready)

5. MODEL DANYCH (CPT)
CPT:
- region
- miejscowosc
- nocleg
- atrakcja
- szlak
- firma
- jezioro
- pogoda
- lead

ACF pola:
- lat / lng
- adres
- udogodnienia
- galeria
- opis
- status pakietu

6. MAP ENGINE
Funkcjonalność:
- mapa Leaflet
- markery (FREE / PREMIUM / PREMIUM+)
- clustering
- filtry (typ, lokalizacja, udogodnienia)
- UX mobile-first

Logika:
- 1 marker FREE na centrum miejscowości
- PREMIUM / PREMIUM+ → indywidualne pinezki
- ranking wg pakietu

7. LEAD ENGINE
Mechanizm:
- formularz kontaktowy przy wizytówce
- maskowanie danych w FREE
- wysyłka zapytania do właściciela
- zapis do CPT lead

Funkcje:
- powiadomienia email
- tracking leadów
- integracja z upsell

8. MODEL PAKIETÓW
FREE:
- brak widocznych danych kontaktowych
- formularz kontaktowy
- niski ranking

PREMIUM:
- pełne dane kontaktowe
- lepsza widoczność
- SEO boost

PREMIUM+:
- najwyższa pozycja
- wyróżnienie na mapie
- limit miejsc na lokalizację

9. CLAIM ENGINE
Proces:
1. wizytówka dodana przez admina (seed)
2. właściciel otrzymuje informację
3. może przejąć wizytówkę
4. weryfikacja (token/email)
5. upsell do PREMIUM

Funkcje:
- claim listing
- automatyczne powiadomienia
- anti-fraud

10. SEEDING STRATEGY
- ręczne dodanie baz danych
- import z Google (opcjonalnie)
- initial data coverage

Cel: osiągnięcie efektu „pełnego portalu” od startu

11. PLANNER (MODUŁ KLUCZOWY)
Cel: narzędzie do planowania wycieczek → konwersja użytkownika

Funkcje:
- wybór atrakcji / noclegów / szlaków
- budowa trasy
- obliczanie dystansu (Haversine)
- kolejność punktów (sorting / drag&drop)
- mapa (Leaflet)

Eksport:
- Google Maps
- zapis planu (future)

Monetyzacja:
- promowane punkty (PREMIUM+)
- CTA → lead
- gotowe trasy (SEO pages)

12. WEATHER ENGINE
- API Open-Meteo
- cache (transients)
- prognoza current + daily
- integracja z lokalizacją (lat/lng)

Rozszerzenia:
- dynamiczny hero (weather-aware)
- sugestie planner

13. UX / UI
Założenia:
- mobile-first
- szybkie filtrowanie
- proste decyzje użytkownika

Elementy:
- mapa + lista
- sticky CTA
- dark mode (opcjonalnie)

14. SEO ENGINE
Strategia:
- programmatic SEO (miejscowości + frazy)
- landing pages:
  - „noclegi + lokalizacja”
  - „atrakcje + lokalizacja”
  - „co robić + lokalizacja”

Content:
- blog (po.beskidzku.pl)
- poradniki
- rankingi

15. MONETYZACJA
Źródła:
- abonamenty (FREE / PREMIUM / PREMIUM+)
- lead generation
- artykuły sponsorowane
- afiliacja (np. booking)
- sponsorowane listingi

16. ROADMAP MVP
Etap 1:
- baza danych (noclegi + atrakcje)
- mapa
- wizytówki

Etap 2:
- lead engine
- pakiety

Etap 3:
- planner
- SEO content

Etap 4:
- skalowanie regionów

17. KLUCZOWE METRYKI
- liczba wizytówek
- liczba leadów
- konwersja FREE → PREMIUM
- ruch SEO
- CTR na mapie

18. DEFINICJA PRODUKTU
Beskidzku.pl to:

lokalny system generowania ruchu i leadów
oparty o dane, mapy i intencję użytkownika,
z monetyzacją w modelu abonamentowym i leadowym.

19. FRONTPAGE v3.0 — SPECYFIKACJA KONWERSJI

CEL STRONY GŁÓWNEJ:
Strona główna to główny punkt wejścia i ścieżka konwersji:

WEJŚCIE → PLAN → WYBÓR → LEAD / BOOKING

SEKCJE GŁÓWNE:

1. HERO (WEJŚCIE GŁÓWNE)
   Cel: natychmiastowe rozpoczęcie planowania

   Zawartość:
   - Nagłówek: "Zaplanuj Beskidy w 30 sekund"
   - Formularz z trzema polami:
     * gdzie? (wybór miejsca)
     * kiedy? (termin/data)
     * z kim? (typ grupy)

   Quick options (filtry szybkie):
   - 👨‍👩‍👧‍👦 z dziećmi
   - 🥾 trekking
   - 🧘 chill
   - 🚴 aktywnie

   CTA: "GENERUJ PLAN"
   Zachowanie: przekierowanie do /planner z parametrami

2. QUICK START (GOTOWE SCENARIUSZE)
   Cel: skrócenie decyzji

   Karty predefiniowane:
   - "Weekend w Beskidach" (2 dni · top trasy + nocleg)
   - "Dla rodzin" (łatwe szlaki + atrakcje dla dzieci)
   - "Najlepsze szlaki" (widokowe + parking)
   - "Jeziora" (plaże, pomosty, aktywności wodne)

   Zachowanie: klik = planner z gotową konfiguracją

3. MAPA + LISTA (DISCOVERY ENGINE)
   Cel: wizualne odkrywanie z filtrowaniem

   Desktop:
   [ LISTA ] | [ MAPA ]

   Mobile:
   [ MAPA ]
   [ bottom sheet → LISTA ]

   Funkcje:
   - filtrowanie: noclegi, atrakcje, szlaki
   - sortowanie: popularność, odległość
   - interakcja: hover = podświetlenie, klik = szczegóły

   Hierarchia markerów (według pakietu):
   - FREE → neutralne szare markery (wspólna lokalizacja)
   - PREMIUM → czerwone markery (dokładna lokalizacja)
   - PREMIUM+ → wyróżnione markery ze specjalnym stylem

4. PLANNER (MODUŁ GŁÓWNY)
   Cel: interaktywne planowanie z punktami monetyzacji

   Struktura:
   - punkt startowy (parking z pojemnością)
   - punkty na trasie/szlaki
   - przystanki przy atrakcjach
   - rekomendacje noclegów

   Wyświetlane dane:
   - dystans (km) — wzór Haversine
   - czas (godziny)
   - profil wysokości (+metry) — do implementacji

   CTA główne:
   - "ZAREZERWUJ" (rezerwacja bezpośrednia)
   - "ZAPYTAJ" (formularz zapytania/lead)

   Eksport:
   - integracja Google Maps (trasa wielopunktowa)
   - zapis lokalny (localStorage)
   - udostępnianie linkiem (przyszłość)

5. NOCLEGI (MONETYZACJA GŁÓWNA)
   Cel: prowadzenie do rezerwacji i leadów

   Elementy karty:
   - zdjęcie
   - nazwa
   - lokalizacja (miejscowość)
   - ocena (0-10)
   - badge (FREE/PREMIUM/PREMIUM+)
   - cena lub "kontakt"

   CTA podwójne:
   - "Zobacz" (pełne szczegóły)
   - "Zapytaj" (formularz leada)

   Logika:
   - FREE → dane kontaktowe ukryte, wymagany formularz
   - PREMIUM → pełny kontakt widoczny
   - PREMIUM+ → najwyższa pozycja, wyróżnienie

6. BLOK POGODY
   Cel: pomoc w planowaniu kontekstowym

   Dane na żywo:
   - aktualna temperatura (°C)
   - status pogody
   - alerty pogodowe

   Przykłady:
   - "21°C — dobre warunki"
   - "Możliwe burze po południu"
   - "Śnieg na szlakach — sprawdź warunki przed wyjazdem"

   Źródło: Open-Meteo API (darmowe)
   Cache: 1 godzina (Next.js ISR)

7. MODUŁ PARKINGÓW
   Cel: wartość praktyczna

   Informacje:
   - nazwa i adres parkingu
   - pojemność (liczba miejsc)
   - typ (darmowy/płatny)
   - odległość do szlaku/atrakcji

   CTA: "Dodaj do planu"

8. GOTOWE PLANY (SEO + QUICK START)
   Cel: landing pages SEO + szybkie planowanie

   Sekcje tras:
   - "1 dzień w Beskidach" (parking → szlak → atrakcja → obiad)
   - "Weekend dla par" (widoki + SPA + kolacja)
   - "Dla rodzin" (łatwe trasy + place zabaw + jeziora)
   - "Top atrakcje [Region]" (najlepsze miejsca w regionie)

   Zachowanie: klik = planner z preloadem
   Wartość SEO: landing pages dla "[aktywność] + [lokalizacja]"

9. ADSENSE (MONETYZACJA DODATKOWA)
   Cel: przychód z reklam bez zakłócania UX

   Umiejscowienie:
   - między sekcjami
   - pod blokami listingów
   - natywny styl dopasowany do designu

   Zasady:
   - nie zakłóca doświadczenia użytkownika
   - niższy priorytet niż CTA lead/booking

   Status: do implementacji (wymaga konta Google AdSense)

10. FINAL CTA (OSTATNIA SZANSA KONWERSJI)
    Cel: reaktywacja przed opuszczeniem strony

    Treść: "Nie trać czasu — zaplanuj teraz"

    Przyciski:
    - Główny: "Zaplanuj" → /planner
    - Dodatkowy: "Znajdź nocleg" → /listings?type=hotel

    Design: czerwone tło, duży biały tekst
    Umiejscowienie: dół strony, przed stopką

11. MOBILE UX
    Cel: optymalizacja dla kciuka

    Elementy:
    - dolna nawigacja (sticky) — do zrobienia
    - duże przyciski dotykowe (min 44x44px)
    - pływający przycisk plannera — do zrobienia
    - układ jednokolumnowy na małych ekranach

    Zasada: wszystkie kluczowe akcje w zasięgu kciuka

12. ŚCIEŻKA KONWERSJI (USER FLOW)

    Kroki:
    1. wejście na stronę główną
    2. interakcja z hero lub quick start
    3. przekierowanie do plannera z kontekstem
    4. dodawanie/edycja punktów wycieczki
    5. przeglądanie noclegów
    6. ocena opcji (cena, lokalizacja)
    7. akcja finalna:
       → formularz leada (FREE), lub
       → rezerwacja (PREMIUM)

    Metryki sukcesu:
    - wskaźnik wypełnienia formularza hero
    - czas w plannerze
    - wskaźnik wysłania leada
    - CTR do noclegów
