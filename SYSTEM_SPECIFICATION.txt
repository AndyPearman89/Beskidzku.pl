1. OVERVIEW
================================================================================

Beskidzku.pl is a regional tourism and business platform serving the Beskidy
mountain region in Poland, operating as:

- SEO-driven content portal (programmatic content generation)
- Local services marketplace (listings, accommodations, attractions)
- Lead generation engine (business inquiries and conversions)
- SaaS layer for businesses (business profiles + monetization)

The system is built as a modular application with scalability to additional
regions.

CORE BUSINESS MODEL:
TRAFFIC → INTENT → LEAD → REVENUE

================================================================================
2. STRATEGIC OBJECTIVES
================================================================================

Build a dominant source of:
- Organic traffic (SEO)
- Local data (accommodations, attractions, businesses)
- Sales leads (inquiries, conversions)

Success metrics:
- Number of business listings
- Number of leads generated
- Conversion rate (FREE → PREMIUM)
- Organic search traffic
- Map click-through rate (CTR)

3. PORTAL STRUCTURE
================================================================================

MAIN SECTIONS:
- Regions (Regiony)

RELATIONSHIPS:

6. MAP ENGINE
================================================================================

TECHNOLOGY:
- Leaflet.js
- OpenStreetMap tiles
- Marker clustering (leaflet.markercluster)

FEATURES:
- Interactive map with custom markers
- Clustering for performance
- Filtering by type, location, amenities
- Mobile-first UX

MARKER LOGIC:
- FREE: 1 marker at town center (shared location)
- PREMIUM: Individual marker at exact location
- PREMIUM+: Individual marker with enhanced icon/styling

MARKER HIERARCHY:
1. PREMIUM+ (highest priority, special icon)
2. PREMIUM (medium priority, standard icon)
3. FREE (lowest priority, generic icon)

MAP INTERACTIONS:
- Click marker → show listing preview
- Filter by category
- Search by location
- Distance calculation (Haversine formula)

================================================================================
7. LEAD GENERATION ENGINE
================================================================================

MECHANISM:
1. Contact form on business profile
2. Data masking for FREE listings (contact hidden)
3. Lead submission to business owner
4. Lead saved to CPT 'lead'

FEATURES:
- Email notifications to business owner
- Lead tracking dashboard
- Conversion tracking
- Upsell prompts (FREE → PREMIUM)

WORKFLOW:
User views listing → Clicks contact → Fills form → Lead sent to owner
→ Lead saved in system → Owner notified → Upsell opportunity

ANTI-SPAM:
- Rate limiting
- CAPTCHA integration (optional)
- Email verification
- Honeypot fields

================================================================================
8. PACKAGE MODEL (MONETIZATION)
================================================================================

FREE TIER:
- Listing visible in directory
- Contact details masked
- Contact via lead form only
- Low ranking in search
- Shared map marker (town center)
- Basic description (limited characters)

PREMIUM TIER:
- Full contact details visible (phone, email, website)
- Individual map marker at exact location
- Higher ranking in search and map
- SEO boost (better indexing)
- Extended description
- Photo gallery (more images)
- Social media links

PREMIUM+ TIER:
- Highest position in listings
- Featured marker on map (special icon/color)
- Homepage carousel placement
- Priority in search results
- Unlimited photos
- Video embedding
- Analytics dashboard
- Limited slots per location (scarcity)

PRICING (INDICATIVE):
- FREE: 0 PLN/month
- PREMIUM: 99 PLN/month or 990 PLN/year
- PREMIUM+: 249 PLN/month or 2490 PLN/year

================================================================================
9. CLAIM ENGINE
================================================================================

PROCESS:
1. Admin seeds listings (initial data)
2. Business owner discovers their listing
3. Owner claims listing via "Claim this business" button
4. Verification via email token
5. Owner gains access to edit listing
6. Upsell flow to PREMIUM/PREMIUM+

VERIFICATION METHODS:
- Email verification (token link)
- Phone verification (SMS code)
- Document upload (business registration)

ANTI-FRAUD MEASURES:
- Email domain matching (business email)
- Manual review for high-value claims
- Rate limiting on claim attempts
- Duplicate claim detection

OWNER DASHBOARD:
- Edit listing details
- View lead history
- Upgrade package
- Analytics (views, clicks, leads)

================================================================================
10. SEEDING STRATEGY
================================================================================

INITIAL DATA SOURCES:
- Manual entry by admin team
- Google Places import (with attribution)
- Public databases (tourism offices)
- User submissions (moderated)

SEEDING PRIORITIES:
1. Major towns (Szczyrk, Wisła, Żywiec, Bielsko-Biała)
2. Top accommodations (hotels, resorts)
3. Key attractions (museums, trails, monuments)
4. Essential businesses (restaurants, shops)

GOAL:
Achieve "complete portal" effect from launch day.
Minimum viable coverage: 100+ listings per major town.

DATA QUALITY:
- Verified addresses and coordinates
- Accurate contact information
- Professional descriptions
- High-quality photos

11. TRIP PLANNER MODULE (KEY FEATURE)
================================================================================

PURPOSE:
Interactive tool for planning trips → drives user engagement → increases
conversion rate.

FEATURES:
- Select attractions, accommodations, trails
- Build custom route
- Calculate distances (Haversine formula)
- Reorder points (drag & drop)
- Interactive map (Leaflet)
- Save and share plans

EXPORT OPTIONS:
- Google Maps export (URL with waypoints)
- PDF download (future)
- Share via link (future)

MONETIZATION:
- Promoted points (PREMIUM+ listings highlighted)
- Call-to-action buttons (contact business)
- Pre-made routes (SEO landing pages)

USER FLOW:
1. User opens planner
2. Searches for places to visit
3. Adds items to itinerary
4. Views route on map
5. Adjusts order
6. Exports to Google Maps or saves
7. Contacts businesses for bookings

TECHNICAL IMPLEMENTATION:
- React component with state management
- Leaflet for map rendering
- Haversine distance calculation
- Drag & drop library (react-beautiful-dnd or @dnd-kit)
- Local storage for draft plans
- API for saving plans (authenticated users)

23. RISKS & MITIGATIONS
================================================================================

RISK: Low listing adoption by businesses
MITIGATION: Admin seeding, proactive outreach, free tier

RISK: Poor data quality
MITIGATION: Verification process, user reports, manual review

RISK: SEO competition (large directories)
MITIGATION: Hyper-local focus, quality content, fast site

RISK: Dependence on Google traffic
MITIGATION: Diversify channels (social, direct, referral)

RISK: Technical scalability
MITIGATION: Modular architecture, caching, CDN, database optimization

RISK: Payment fraud
MITIGATION: Stripe fraud detection, manual review, KYC
